## 2. 已知取舍与风险登记（有意为之，改前先看）

1. **趟次编号漂移**：编号按发车年动态计算，补录旧账会让后面所有趟号后移。取舍：编号语义纯净（第 N 趟就是当年第 N 次发车）> 编号恒定。若用户反馈困惑，方案是详情页展示"编号会随补录变化"说明，而非持久化编号。
2. **双开覆盖**：同浏览器开两个标签同时写账，后保存者覆盖先保存者（无跨标签同步）。个人单机使用场景风险极低，不为此上 BroadcastChannel。
3. **file:// 存储易失**：部分安卓浏览器对 file:// 页面的 localStorage 隔离/清理策略激进。缓解手段已上线（备份提醒横幅 + 保存失败警告）；根治方案是托管到 GitHub Pages（见 §3），但**迁移到 https 域名等于换存储原点，必须先导出备份再在新地址恢复**。
4. **弹层开着时刷新**：残留的历史格会让刷新后第一次系统返回"没反应"（空消耗一格）。无害，不修（unload 钩子代价大于收益）。
5. **iOS 边缘手势理论双触发**：系统侧滑返回与应用自实现左缘右滑可能同时生效。实测中系统手势会 touchcancel 掉页面手势，未见双关；若出现，给 initEdgeSwipe 补 touchcancel 处理即可。

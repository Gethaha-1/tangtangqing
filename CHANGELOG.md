# 变更记录（CHANGELOG）

## v1.2.0 · 2026-07-14

### 视觉（全新双主题）
- 日间主题重做为「宣纸账房」：宣纸底、朱砂主色、文书双线框主卡、区块朱记、宋体展示数字（双主题统一）；
- 夜间主题重做为「瓷青泥金」：瓷青纸底、泥金主色，替代旧"夜路仪表盘"（同宗纸墨意象 + 夜路低眩光）；
- 收讫章升级双圈样式，双主题常显；移除夜间 LED 边条与仪表刻度贴片；
- 底部导航、主题按钮、滑动收车箭头全部由 emoji 换为线性 SVG 图标（科目 emoji 属用户内容，保留）；
- 背景氛围渐变（画在 body::before 固定层，规避 iOS background-attachment 问题）。

### 交互
- **系统返回键接管**：安卓返回键/手势现在逐层关闭弹层，而不是退出页面（History API + expectPop 计数，详见 ARCHITECTURE §6）；
- 快记支持**改日期**（默认今天，折叠控件，一次点开）；
- 改一笔支出时可**顺手改科目**（金额盘顶部科目 chips）；
- 收车时**零收入确认**：没填收入会先问一句，防止忘填；
- 全局**防连点**（600ms 提交锁）与**防穿透**（关层 350ms 触摸挡板）；
- 键盘对尾随 `+`（如 `300+`）容错，不再误报"先输入金额"。

### 数据安全
- 保存失败即时 toast 警告并引导导出备份；
- 首页新增**备份提醒横幅**：从未备份或超 30 天未备份时出现，点击直接导出；导出成功记录 `settings.lastBackupAt`；
- `migrate()` 增加导入数据清洗：金额强转数字（坏值归 0）、缺失数组补齐——恢复来历不明的备份文件也不会污染统计。

### 修复
- 收车结算页：有金额的**停用收入科目**现在会显示（此前金额计入利润却不可见，与详情页口径不一致）；
- 首页仪表盘：支出占比 >100% 时文字显示真实百分比（此前被截在 100% 产生误导），进度条最多画满；
- 首页脚注版本号错标 v1.0 → 版本号统一由 `APP_VER` 常量渲染（单一来源）；
- `openAmtPad` 的 `okLabel` 配置此前从未生效 → 键盘工厂新增 `setOk()`，按钮文案随场景变化（✓ 保 存 / ✓ 记 上 / ✓ 填 好）；
- `fmtMD` 中的无效 `replace('-','-')` 空操作清理；
- 原 `closeTopSheet` 死代码被历史栈机制正式启用。

### 工程化
- 新增 **[JS-6] 内置自检**：21 项断言覆盖统计口径、迁移清洗、金额计算与转义；地址加 `#test` 或控制台 `runSelfTests()` 运行，结果弹窗展示（复用报告弹层）；口径测试用夹具数据、测后还原；
- 建立完整文档体系：README / ARCHITECTURE / DESIGN / TESTING / AI-GUIDE / CHANGELOG / BACKLOG；
- 应用文件更名为 `index.html`（利于 git 固定文件名与 GitHub Pages 直接托管）。

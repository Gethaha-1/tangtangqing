# 测试（TESTING）

适用版本：v1.2.0。单文件零依赖的工程化方案：**内置自检 + 手工冒烟清单**。每次改代码，两样都要过。

## 1. 内置自检（runSelfTests，[JS-6]）

### 怎么跑

- 浏览器打开 `index.html`，在地址末尾加 `#test` 回车刷新 → 弹出「自检报告」，逐条列出 ✓/✗；
- 或打开控制台执行 `runSelfTests()`（返回 true/false，同时 console 打印全文）。

### 覆盖范围（21 项断言）

| 类别 | 覆盖点 |
|---|---|
| 纯函数 | evalExpr 连加/小数/空串/拒绝负数；fmtMD；yearOf/monthOf；esc 防 XSS |
| 迁移与清洗 | 字符串金额洗成数字；缺失数组补齐；空数据回默认；settings 新字段默认值合并 |
| 统计口径 | 单趟利润；编号按发车年、跨年重新编号；利润按到家日期归年（含跨年趟）；维修单列；月度合计；常用金额频次排序；天数含首尾 |

### 设计原则（加测试前必读）

1. **不碰真实数据**：口径类测试把全局 `S` 换成夹具（fixture），`finally` 里必须还原；
2. **只测纯逻辑**，不测 DOM 渲染——渲染靠手工冒烟；
3. 夹具里特意放了一个**跨年趟**（2025-12-28 发车、2026-01-03 到家），它是两条口径（编号按发车年、利润按到家年）的分界样本，别删；
4. 加断言：在 `runSelfTests` 里按 `eq(实际值, 期望值, '中文说明')` 追加即可，`eq` 用 JSON 深比较。

### 什么时候必须跑

改了 [JS-1]/[JS-2] 的任何函数 → 必跑；改口径相关 → 必须先加断言再改代码（测试先行）；其他改动 → 提交前跑一遍兜底。

## 2. 手工冒烟清单（改完过一遍，手机浏览器优先）

1. 发车 → 首页出现"当前趟"卡片，再点发车被拦截；
2. 记一笔：选科目 → 输 `300+250` 显示合计 → 记上；点"记在今天"改成昨天再记一笔，日期正确；
3. 快速连点"✓ 记 上"两次 → 只入账一笔（防连点）；
4. 收车：不填收入直接滑 → 弹"还没填收入"确认，选"再想想"滑块回弹；填上收入再滑 → 收车成功、趟次卡出现收讫章；
5. 趟次详情：改一笔支出的金额+科目+日期；删一笔；补一笔收入；
6. 系统返回键：开详情→开改一笔（两层弹层），按两次系统返回逐层关闭，第三次才退页面；
7. 补录一趟旧账，趟次列表编号排序正确；
8. 科目：新建→改名→停用；停用后快记里不再出现，但历史账目仍显示；有金额的停用收入科目在收车页仍显示；
9. 报告：生成上月报和年报，复制全文成功；
10. 备份：导出 → 首页备份横幅消失；清数据后恢复备份 → 账目完整；
11. 切主题往返，重进应用主题被记住；仪表盘在支出>收入时文字显示 >100%、条画满；
12. 地址加 `#test` → 自检全绿。

## 3. 兼容性检查点

改动涉及手势 / History / 剪贴板 / 分享时，至少验证：iPhone Safari、安卓 Chrome、微信内置浏览器（用户最常从微信收藏打开）。已知：微信内置浏览器里 Web Share 可能不可用（代码已降级为复制）。

# 设计系统（DESIGN）

适用版本：v1.2.0。改任何视觉之前先读这份，保持气质统一。

## 1. 设计语言：一对同宗的中式纸墨主题

- **日间 · 宣纸账房**：宣纸底、朱砂笔、文书双线框、宋体大数、楷体"收讫"章。意象来自老账房先生的纸账本——目标用户从纸本子迁移过来，视觉上要"认识它"。
- **夜间 · 瓷青泥金**：瓷青纸底、泥金数字。意象来自古籍中的瓷青纸泥金写经（深青黑纸配金墨）。选它不只为好看：深底 + 暖金是夜路驾驶室里眩光最低的组合，替代了旧版"夜路仪表盘"的琥珀色方案，且与日间主题同属一个文化谱系。

两主题共享同一套变量名，切换只翻转 `body` 的 class。**新增颜色一律走变量，不许写死。**

## 3. 字体

- 正文：系统黑体栈（-apple-system / PingFang SC…）；
- 展示数字与品牌（`.display`）：宋体栈 `"Songti SC","Noto Serif CJK SC","SimSun",serif`——双主题统一，日间是朱砂宋体、夜间是泥金宋体；
- 图章（`.hand`）：楷体栈。
- **不引外部字体**，全部系统字体，保证单文件离线。

## 4. 标志性组件（signature）

1. **文书双线框**（`.panel.hero::after`）：首页主卡外框 + 内框 7px 双线，日间金褐、夜间淡金。只用在主卡上，别到处加。
2. **收讫章**（`.stamp`）：已收车趟次卡右上角，朱砂双圈楷体章，旋转 -12°，双主题常显。
3. **朱记区块标**（`.sec-label::before`）：区块标题前 4×13px 的主色竖记。
4. **滑动收车**（`.slide-close`）：结账级操作必须滑动确认，不许改成点按。

## 5. 图标规范

- 界面图标（导航、主题钮、滑块箭头）：**线性 SVG**，viewBox 24、stroke 1.8（滑块 2.0）、round 端点、`currentColor` 取色，字符串形式内联（见 [JS-2] 的 SVG_* 常量与 nav 的内联 SVG）；
- 科目图标：保留 emoji——它们是**用户内容**（用户建科目时选的），不属于界面装饰；
- 横幅前缀（📄🔒⚠️）保留 emoji：这是提醒语气的一部分，面向不熟悉图标语言的用户。

## 6. 文案语气（写给司机，不写给程序员）

动词开头、口语、不用术语。对照表：

| 别写 | 要写 |
|---|---|
| 取消 | 再想想 |
| 确认提交 | ✓ 记 上 / ✓ 填 好 |
| 数据导出成功 | 已导出备份，发到自己微信收藏最保险 |
| 请输入金额 | 先输入金额 |
| 交易分类 | 钱都花哪了 |

规则：按钮说清"按了会发生什么"；操作全程用同一个词（发车→收车，不混用"结束行程"）；空状态是行动指引不是抱歉；错误提示讲怎么补救。

## 7. 可及性底线

- 金额、按钮字号不小于现状（用户可能是老花眼）；
- 数字一律 `tabular-nums`（`.num`）对齐；
- 次要文字对比度不再调浅；
- 已登记待办：放开 `user-scalable=no` 允许双指缩放（BACKLOG #9）。
